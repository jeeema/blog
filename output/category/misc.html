About the basic uses and several small things of reST

Inline markers

The right-hand forms are called "interpreted text roles". The semantics for this is like this; :rolename:text

reST:

*emphasis* = :emphasis:`emphasis`

**strong emphasis**  = :strong:`strong emphasis`

``literal``  = :literal:`literal`

:superscript:`superscript` :subscript:`subscript`

This will look like this:

emphasis = emphasis

strong emphasis = strong emphasis

literal = literal

superscript subscript

Lists

Normal lists(including the numbered and the non-numbered)

reST code:

* ELEMENT1
* ELEMENT1

#. ELEMENT1
4. ELEMENT1

     1. element1
     #. element2

#. ELEMENT1
#. ELEMENT1

This will look like this:

#. ELEMENT1 2. ELEMENT2 #. ELEMENT3

  1. element1
  2. element2
  1. ELEMENT4
  2. ELEMENT5

Definition lists

reST code:

term1
    description of term1
term2
    description of term2

term3
    description of term3

This will look like this:

term1
description of term1
term2
description of term2
term3
description of term3

The blank line between term2 and term3 is removed. It seems the blank lines between blocks which have the same indents are removed.

Literal blocks

The blocks after literal block markers will be unindented, and the font of each block will be a monospaced font, in all cases. There must be more than one line between a line containing the marker between the literal block.

If the markers occur in a line containing a non-space character, single colon following that non-space character will be displayed:

The next paragraph is a code sample::

    def func():
        print("Hello World")

So this will look like this:

The next paragraph is a code sample:

def func():
    print("Hello World")

If the markers occur following a space, the colons are not displayed:

The next paragraph is a code sample.
::

     def func():
         print("Hello World")

So this will look like this:

The next paragraph is a code sample.

def func():
    print("Hello World")
